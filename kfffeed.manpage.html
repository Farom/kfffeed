Name

kfffeed — Export your KDE Addressbook for use on AVM FRITZ!Box Fon WLAN 7270

Synopsis

kfffeed [-cc code] [-ac code] [-pn name] [-o file] [ -nn file] [ -w]

kfffeed [ [ -cc | --country-code ] Country Code ]

kfffeed [ [ -ac | --area-code ] Area Code ]

kfffeed [ [ -pn | --phonebook-name ] name ]

kfffeed [ [ -o | --output-file ] filename ]

kfffeed [ [ -nn | --netnumbers-file ] filename ]

kfffeed [ -w | --write-to-config ]

kfffeed [ --help ]

Description

Kfffeed is a tool for exporting your contacts from Akonadi/ Kaddressbook/Kontact to a AVM FRITZ!Box Fon WLAN 7270.

The AVM FRITZ!Box Fon WLAN 7270 has a built in Dect base station. All phones registered there can have an own Phonebook. kfffeed exports akonadi contacts to these phonebook xml file, which then can be imported by webfrontend of the FritzBox.

kfffeed has to know the phonenumber of your AVM FRITZ!Box Fon WLAN 7270. If you have the international number +49 12345 6789 and have a friend with number +49 12345 5432. So you would save this number on your computer in kaddressbook. When he calls you and you have the same number saved in you Fritz!Box, it gets the number 5432. So it does not recognize the caller as your friend and just shows the callernumber.

If you live in a country, where you always have to dial Area Codes before the number (so local numbers can begin with a 0) please write an email to me. At the moment Area Codes are recognized if they begin with 0 and not 00, so i would program some localization support for this.

Options

--help

Display help and exit.

-cc , --country-code Country Code

Country code of the country where your Fritz!Box is standing. For Germany it is 49. In this Version only germany is supported. You just have to edit the netname.xml to support your country. Send lists so i will generate XML-files for you.

-ac , --area-code Area Code

Area code of the city where your Fritz!Box is standing. For Berlin in Germany it is 30. If you find an Area Code that is not known by kfffeed, send it to me.

-pn , --phonebook-name Phonebook Name

Specify the name you see on the Fritz Fon. It is the first line you see pressing dictionary

-o, --output-file output-file

Specify output file to write to instead of KAddressbook-Fritz-Box-Addressbook.xml

-nn, --netnumbers-file netnumbers-xml-file

Specify netnumbers file to read from instead of netnumbers.xml in local directory.

-w, --write-to-config

Setting this parameter makes kfffeed writing all options to a configfile. Usually this is $HOME/.kde/share/config/kfffeedrc. No data is exported in this case. Called first time it creates the configfile with defaults overwritten by commondline.

Examples

Example 1. Fast

./kfffeed -cc 49 -ac 30 -nn /home/user/src/kfffeed/netnumbers.xml -o export.xml

I would start this, if my fritzbox stands in my flat in Germany/Berlin with the phonenumber +49 30 12345678


Example 2. Doing the same and save config

./kfffeed -cc 49 -ac 30 -nn /home/user/src/kfffeed/netnumbers.xml -o export.xml -w
./kfffeed

This exports the same as above. First the config is written. Then the export is done.


FILES

./kfffeed

The executable.

netnumbers.xml

kfffeed uses a XML-file that describes all phonenets on earth. Per default it has the name netnumbers.xml. It has a simple schema and ‘Yes’ it is big. It is slow too and needs lots of ram. But it knows, where a number comes from so it is possible to print numbers pretty and some other nice features for the future. At the moment germany is built in. If you have a list of all area codes of your country (with citynames), send it to me. Too it gives the feature to check, whether phonenumbers are equal. Will need this for future versions.

phonenet.xsd

kfffeed uses a XML-file that describes all phonenets on earth. This file is a schema that describes this file. If you want to edit netnumbers.xml you can check against this file for correct editing. For all other things, it is useless.

$HOME/.kde/share/config/kfffeedrc

kfffeed needs some configurations, which will not change mostly. So you can save options to this file. [kfffeed] area-code=30 country-code=49 netnumbers-file=/home/user/src/kfffeed/data/netnumbers.xml output-file=export.xml phonebook-name=Peters Telefonbuch

Exit Codes

0

Everything was OK

1

netnumbers.xml was not found. Check config files or set -nn option.

2

Environment was not set carefully

3

Outputfile could not be written.
